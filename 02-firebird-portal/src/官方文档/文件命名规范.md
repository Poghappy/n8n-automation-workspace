# 🏷️ 火鸟知识库文件命名规范

## 📋 命名原则

### 1. 简洁性原则
- 文件名长度控制在30个字符以内
- 避免冗余描述词汇
- 使用核心关键词

### 2. 一致性原则
- 统一使用下划线分隔符
- 统一前缀命名规则
- 统一文件类型标识

### 3. 可读性原则
- 文件名能直接反映内容
- 避免缩写和代码
- 使用标准术语

## 🎯 命名规范

### 核心文档命名格式
```
[模块]_[类型]_[功能].md
```

### 前缀规范
- `火鸟_` - 火鸟系统相关文档
- `API_` - API接口文档
- `GPTs_` - GPTs相关文档
- `用户_` - 用户指南文档
- `开发_` - 开发文档

### 类型标识
- `接口` - API接口文档
- `指南` - 使用指南
- `规范` - 标准规范
- `配置` - 配置文档
- `教程` - 教程文档
